# Non-Digital-Native UX Guidelines

## 結論
kamio-ai をデジタル非ネイティブ層に使ってもらうには、
- 画面の複雑さを削る
- 操作導線を固定する
- 人が伴走する前提を組み込む
この3点が必須。

## 想定ユーザー
- 高齢者
- IT業務に不慣れな店舗オーナー
- スマホは使えるが、設定や手続きに不安がある人

## UX設計の原則
1. 1画面1目的
- 画面ごとに「何をすればいいか」を1つに絞る

2. 文字でなく行動を示す
- 「登録する」「相談する」「次へ」のように動詞で表示

3. 専門語を使わない
- PoC / KPI / DAO などは初期画面で使わない

4. 選択肢は3つ以内
- 選択疲れを防ぐ

5. 失敗しにくい設計
- 戻るボタン固定
- 入力エラーはその場で赤字表示

6. 常に人へつながる導線
- `電話` `メール` `説明会` の3導線を常時表示

## UI要件（最低限）
- 文字サイズ: 16px以上
- 行間: 1.6以上
- ボタン高さ: 48px以上
- CTAは画面下部に固定
- コントラスト比を確保（薄いグレー文字は避ける）

## 推奨画面フロー
1. 入口
- 選択肢: `参加したい` / `まず相談したい` / `資料だけ見たい`

2. 参加案内
- 3ステップで説明
- 所要時間を明記（例: 3分）

3. 申込
- 必須項目を最小化
- 入力例を表示

4. 完了
- 次の行動を明記
- 連絡先を再表示

## 支援運用（UIだけで解決しない前提）
- 30分のオンライン説明会を定期開催
- 店舗向けに「初期設定代行」を用意
- よくある質問を紙1枚でも配布

## NG
- 初期画面に長文を置く
- 専門語を連発する
- 1画面で複数判断を求める
- フォーム入力だけで完結させる

## まず実装する改善
1. LPのCTAを3分類（参加/相談/資料）で整理
2. Google Form項目を最小化（必須は氏名・連絡先・区分）
3. 「電話相談希望」選択を追加
4. 完了画面で次アクションを明示
